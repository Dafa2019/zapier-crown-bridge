When a new production Sentry issue appears for project zapier-crown-bridge:

1. Inspect the stack trace, related commits, and directly relevant code paths.
2. Determine the most likely root cause.
3. Propose the smallest safe fix.
4. Run relevant tests.
5. Open a pull request only if confidence is high and the change is bounded.
6. In the PR summary include:
   - issue link
   - root cause
   - fix summary
   - risk assessment
   - rollback note
7. Post the PR link to #engineering.
